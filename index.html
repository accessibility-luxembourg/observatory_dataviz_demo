Digital accessibility Luxembourg data visualization components

Welcome

This page presents six diagrams created in CSS using semantic and accessible content, coded using only the following HTML tags:

  • <table>
  • <dl>, <dt>, <dd>
  • <ul>, <li>

These HTML elements are then “repainted” using CSS.

This project comprises four files:

  • index.html: the content;
  • main.css: a few elements to style the page and, optionally, the modules (tables, lists and definition lists) when the modules.css file is disabled;
  • charts.css: the style sheet that performs transformations on tables, lists and definition lists, turning them into infographics;
  • main.js: an optional script that allows you to enable/disable the charts.css style sheet and animates the infographics when they enter the viewport. This feature, which uses the IntersectionObserver API, may eventually be replaced by the CSS animation-trigger property once it is supported by all browsers. If you prefer to do without it, simply ignore the .hideChart and .animChart statements.

This project is open source, licensed under CC0, and therefore freely reusable.

These visualisations were tested as part of an accessibility audit carried out in late 2025. No issues were identified, in line with the Observatory’s accessibility statement.

These visualisations are currently active on the Digital accessibility observatory.

You can get access to the code on our GitHub repo.

They are inspired by the work of Gaël Poupard, a front-end developer based in Nantes. We invite you to visit his page and, more specifically, his Chaarts project for further information.

Definition list to column chart

Here, the structure of the dd list is transformed into a column chart. The height of the bars is defined using the style attribute and a --value variable. The difference in style for the <dd> attribute when the value is zero is achieved using a specific .low class.

Table to pie chart

For the purposes of the observatory, only two elements are required: the proportion with a statement, and the proportion without a statement. The table is redrawn as a legend, and we use the ::before and ::after pseudo-classes to create the two sections of the pie chart. ::before creates the entire pie chart, whilst ::after places the other section, using the CSS property background: conic-gradient, which incorporates the white edges.
